Maturity Rating Scale

Level Rating Description
0 Non-existent No capability or practice in place
1 Initial Ad-hoc, inconsistent, manual processes
2 Developing Defined processes, partial coverage
3 Established Consistent execution, good coverage
4 Optimizing Metrics-driven, continuous improvement

Interpretation: Two-Dimensional Maturity

High Process + High Tools
Mature AppSec program. Focus on optimization and benchmarking.
High Process + Low Tools
Good foundations. Invest in tooling to automate processes.
Low Process + High Tools
Tool-heavy but process-poor. Develop governance and training.
Low Process + Low Tools
Early stage. Start with foundational processes before tools.

How to Use These Forms

  1. Workshop setting: Conduct with AppSec team, development leads, and security architects
  2. Rate each item: Score 0-4 based on the maturity scale
  3. Document evidence: Note specific examples supporting each rating
  4. Calculate totals: Sum scores for each dimension (/32 each)
  5. Plot on quadrant: Identify which quadrant your organization falls into
  6. Create roadmap: Prioritize improvements based on gaps and RMIT requirements

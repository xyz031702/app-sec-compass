PCI-DSS Card Skimmer Demo

Purpose: Demonstrate card skimming attacks and protection mechanisms for PCI-DSS requirements 6.4.1 and 11.6.1

1. Card Number Skimmer Attacks

Via Vulnerable jQuery 3.3.1

  1. Run clear.html
  2. Run payment_jquery.html to show no injection
  3. Run seed.html to inject
  4. Run payment_jquery.html to show injection

Via XSS

  1. Run clear.html
  2. Run payment.html to show no injection
  3. Run seed.html to inject
  4. Run payment.html to show injection

2. CSP and SRI Protection

  1. Go into csp-sri folder
  2. Run demo.sh to start local server
  3. Click seed.html to inject, check injected content: console → application → localStorage → localhost:8080
  4. Click payment_vulnerable.html to show injection
  5. Click payment_protected.html to show no injection, check CSP block error in console
  6. Click csp-sri-guide.html to show the guide

3. CSP and SRI Limitations

When CSP and SRI Cannot Block All Attacks

  • Go into jquery_failed folder
  • Open index.html
  • Open console to show the DOM-XSS attack
  • Enter something to show card number is still skimmed

CSP/SRI Limitations

  • Some libraries don't provide SRI hashes (e.g., Stripe.js): GitHub Issue #167
  • Rare case: Corporate SSL inspection can intercept and modify content (requires pre-configured trust)
  • Rare case: CDN/infrastructure compromises at the source can serve malicious code from trusted domains

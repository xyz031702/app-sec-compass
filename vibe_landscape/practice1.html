Understanding Prompt-Based Architecting

Prompt-based architecting is an emerging approach to software design where developers use natural language prompts to generate code, architectural diagrams, and system designs. This practice leverages large language models (LLMs) to translate high-level descriptions into implementable solutions.

While this approach can dramatically accelerate the design phase and reduce initial development effort, it also introduces significant risks related to hallucinated features, incorrect assumptions, and over-reliance on AI-generated solutions without proper verification.

Detailed Example

The Hallucinated Library Problem

A developer at a fintech startup was tasked with creating a data visualization dashboard for transaction analytics. Under time pressure, they prompted an AI: "Create a data visualization library for financial transactions with interactive charts."

The AI confidently generated code that relied on a library called graphi-py, which it described as "a popular financial visualization library." The code looked impressive and comprehensive.

The developer spent three days trying to install and configure this library before realizing it didn't exist—it was completely hallucinated by the AI. This wasted valuable development time and created frustration.

When they finally switched to a real library (Plotly), they had to rewrite significant portions of the code, as the AI-generated architecture was designed around the fictional library's API.

Risk Details

  • Hallucinated Dependencies

    AI models may reference non-existent libraries, functions, or APIs, leading to significant debugging time and architectural rework.

  • Architectural Misalignment

    Generated architectures may not align with company standards, leading to inconsistent codebases and increased maintenance costs.

  • Security Blind Spots

    AI may not consider organization-specific security requirements, potentially introducing vulnerabilities that standard security reviews might miss.

  • Over-Complexity

    AI tends to generate more complex solutions than necessary, creating maintenance challenges and potential performance issues.

Recommended Guardrails

  • Use Prompt Templates

    Create standardized prompt templates that require specifying libraries and versions to be used.

  • Test-First Mindset

    Write tests before generating code to ensure the AI's output meets functional requirements.

  • Dependency Verification

    Automatically verify all libraries and dependencies referenced in AI-generated code.

  • Architecture Reviews

    Require senior developer review of AI-generated architectures before implementation begins.
